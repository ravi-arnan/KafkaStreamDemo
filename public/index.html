Konsep Utama Apache Kafka

Broker
Server pusat Kafka yang menerima, menyimpan, dan melayani message. Satu cluster bisa memiliki banyak broker untuk High Availability dan fault tolerance.
Topic
Kategori atau "saluran" tempat message dipublish. Seperti nama folder atau channel. Satu topic dibagi ke beberapa partisi untuk distribusi beban.
Partition
Unit pembagi horizontal sebuah topic. Message ditulis ke partisi tertentu berdasarkan partition key. Memungkinkan pemrosesan paralel dan horizontal scaling.
Offset
Penanda posisi urutan message dalam partisi. Consumer menyimpan offset terakhir yang diproses. Mendukung replay data dari offset mana saja bila dibutuhkan.
Producer
Aplikasi yang mempublish message ke topic Kafka. Bisa menentukan partisi tujuan dengan partition key. Mendukung batching dan kompresi untuk performa tinggi.
Consumer Group
Kumpulan consumer yang bekerja bersama membaca satu topic. Setiap partisi hanya dibaca oleh satu consumer per group — memungkinkan load balancing otomatis.
